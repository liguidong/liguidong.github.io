VirtualBox+vagrant+Homestead 安装laravel环境

2018-01-07

VirtualBox+vagrant+Homestead 安装laravel环境

win上的配置，其他平台请移步
本地安装配置laravel的环境多种多样

常见的有win上集成环境安装使用，linux虚拟机安装使用

而linux虚拟机上安装根据环境又有多种方式：编译安装，vagrant，docker 等

编译安装不赘述，vagrant及docker的区别，引用知乎的一段话来说明

作者：cikenerd

 

本质区别Vagrant并不提供虚拟化技术，本质上是一个虚拟机外挂，通过虚拟机的管理接口来管理虚拟机，让用户更轻松的进行一些常用配置，比如：CPU/Memory/IP/DISK等分配。并且提供了一些其它的管理操作：比如开机运行指定命令，镜像二次打包，插件编写等等。vagrant官方有介绍

To achieve its magic, Vagrant stands on the shoulders of giants. Machines are provisioned on top of VirtualBox, VMware, AWS, or any other provider. Then, industry-standard provisioning tools such as shell scripts, Chef, or Puppet, can be used to automatically install and configure software on the machine.

而docker是一个容器引擎，每一个实例是一个相对隔离的空间，与宿主机共享操作系统内核，并且共享宿主机资源。相对于披着虚拟机皮的vagrant，docker更加轻量，消耗更少的资源。贴一张docker官方介绍图

过程综述

1.安装VirtualBox

2.安装vagrant

3.下载Homestead box

4.下载Homestead并配置

5.启动vagrant

6.ssh连接并安装laravel

ps：建议在git Bash下进行操作，cmd可能会出问题（如无git建议下载安装，全程依然傻瓜式下一步）

下载地址：http://gitforwindows.org/

一、安装VirtualBox

https://www.virtualbox.org/ 下载安装，傻瓜式

二、安装Vagrant

https://www.vagrantup.com/ 依然傻瓜式下载安装

三、下载Homestead box

可在线，可离线下载进行，因天朝网络环境可能失败，上梯子无效，不过个人还是在线下载下来了

$ vagrant box add laravel/homestead
打开git bash 输入以上，弹出选项，选择virtualbox，即可自动开始下载

如果下载进行不下去，可复制其中的链接用IDM，迅雷等工具进行下载

https://atlas.hashicorp.com/laravel/boxes/homestead/versions/5.0.1/providers/virtualbox.box


挖坑，如果是离线下载的box，需要做些其他的事情，因我这边未碰到，所以暂时留坑


此时，C盘用户文件夹中，应该多了此文件夹 .vagrant.d,下载的box即在其中

下载Homestead并配置

在Git bash 里输入

$ git clone https://github.com/laravel/homestead.git ~/Homestead

这里下载好后继续

# 先cd到刚才下载的Homestead文件夹

$ cd ~/Homestead

# 开始初始化Homestead

$ bash init.sh

此时，在用户目录下 Homestead文件夹中，可以看到多了一个Homestead.yaml文件

他默认应该是介个样子，其中有几条配置需要了解，处理一下（这个配置很重要，下面所做的几个操作都是围绕他进行）

首先我们需要建立SSH Key，在Git bash中输入

$ ssh-keygen -t rsa

期间一路回车就好，用户名密码都暂为空

完事后，我们会在用户目录看到一个.ssh目录

里面的两个文件即对应配置中的这几行

authorize: ~/.ssh/id_rsa.pub
​
keys:
    ~/.ssh/id_rsa
​

接下来，看配置文件这里

folders:
    - map: ~/Code
      to: /home/vagrant/Code
​
sites:
    - map: homestead.test
      to: /home/vagrant/Code/laravel/public

其中folders中，将map后面的文件夹（即~/Code）映射到Homestead中的/home/vagrant/Code。这就像我们常见到的和虚拟机中系统共享文件夹类似，我们把本机的~/Code文件夹分享给Homestead，所以我们可以很方便地使用自己喜欢的IDE（比如PhpStorm）进行开发，在~/Code中所进行的修改会如实反应在Homestead对应的文件夹中。

在sites中，我们定义了homestead.test指向/home/vagrant/Code/laravel/public这个文件夹，即Laravel项目的public文件夹。这样在浏览器中输入homestead.app，我们就可以直接看到项目主页了。

注意：以上的文件夹及映射是可以根据用户喜好更改的，比如这里我将开发文件夹叫做WebDev，里面我新建了一个叫teahouse的项目，那么以上的配置，我就可以做出相应修改：

folders:
    - map: ~/code
      to: /home/vagrant/Webdev
​
sites:
    - map: teahouse.app
      to: /home/vagrant/Webdev/teahouse/public

之后，为了teahouse.app这个域名能正常的访问，我们需要在hosts中增加一行

192.168.10.10 homestead.app teahouse.app

ps:hosts位于C:\Windows\System32\drivers\etc
再接下来我们看配置文件中

databases:
    - homestead

在这里我们没添加一个数据库名字，Homestead就会创建一个数据库

这里我添加了一个新的数据库

databases:
    - homestead
    - teahouse

注意不要使用Tab来缩进，可能会报错，这里使用空格

接下来，我们就可以来启动Vagrant了，惊不惊喜！

Git bash中输入

cd ~/Homestead

vagrant up

过会儿，我们就能看到Vagrant已经顺利运行了，接下来我们可以通过本地的ssh工具，mysql工具来连接环境

默认的SSH：

Host：127.0.0.1

端口：2222

用户：vagrant

密码：vagrant
默认MySqlhomestead/secret

SSH连接上后，接下来按照之前配置中的路径来新建服务器上的目录

例如我的配置是如此这般：

sites:
    - map: teahouse.app
      to: /home/vagrant/Webdev/teahouse/public

我于是在服务器端新建如下目录（teahouse/public 这里已经是laravel里面了）

进入/home/vagrant/Webdev目录，接下来通过composer安装laravel

出于GFW原因，我们不得不用中国镜像来安装。vagrant虚拟机中运行如下命令

composer config -g repo.packagist composer https://packagist.phpcomposer.com
接着我们通过composer来安装一个5.5版本
composer create-project laravel/laravel=5.5 –prefer-dist
下载完成后，Webdev下出现laravel目录，接着将目录名改为配置中自己的名字，这里我配置的是teahouse
mv laravel/ teahouse/
接下来本地通过配置的域名进行访问，出现laravel界面，ok，大功告成

我为什么不再刷机

2017-05-22

一、前言

记得在某厂家ROM内测时，我填过这样一段申请

从8250到8064到810再到820

从四叶草到miui到flyme再到氢os

从HVGA到WVGA到HD再到FHD

从甜甜圈到姜饼到果冻豆再到棒棒糖

从高中到大学直至工作，我眼见着这最触手可及的电子设备变化翻天，希望我的经历能帮助*继续成长…

11年6月，第一次拥有自己的安卓设备（1.6），那一晚我兴奋到半夜，此后的两三年中，换手机不勤（穷学生），但每个月起码的几次刷机，却几乎从未间断，但时至今日，我却很难提起折腾的兴趣，坐在电脑前，看着曾经热闹，如今发帖寥寥的社区，论坛，只能感叹，环境（生态）变了，我，也变了

二、回顾

11年暑假，丝毫见不到阳光的两个月，手中托盘盛满酒肉，行走在射灯照耀的走廊中，眼见一桌桌客人推杯换盏，抽得闲暇，找个角落，掏出手机，手中的乐Phone W101 这就是两个月来坚持的意义，上班没多久就预支工资，将他拿到了手。

高通骁龙8250 1GHz

512RAM

512ROM

后置500w 支持自动对焦

这在11年，配置简直豪华，也是他，让我知道了lephone之家（现在已不在了），知道了机锋，知道了什么是root，体验不佳的四叶草UI也逼我知道了刷机是干嘛，也是他 让我发现原来还有miui如此良心的rom（当时rom市场环境不佳，几乎无优质本地化rom）

让我知道了cm，等国外rom（当时毫无好感，丑哭的界面外加丝毫没有本地化的功能）

第二台机器：MOTO Defy（ME525）

土豪朋友入手了当时火的一塌糊涂的戴妃，又看上了我手里的乐Phone，两厢情愿的情况下，换机

德仪3210 800mHz（ARM A8）

512RAM

后置500w 自动对焦

纸面参数来看，似乎与我手机并无多大区别，但实质3610 采用 45nm制程，比8650的60nm不知高到哪里去了，支持720p视频录制即是铁证（需刷里程碑2移植rom），同时支持防水防尘防刮（IP57+第一代康宁大猩猩玻璃），不知道多少小伙伴垂涎三尺。

正是这台机器，让我知道了超频，知道了换内核，知道了什么是移植rom，知道了BL锁，知道了同一型号也会有硬件差别（当时市售defy 按摄像头镀膜颜色分为红头，绿头，金头，按BL锁等级又分为4,5,6,7等），也同时让我体验到了变砖的刺激，请假网吧刷机的燃烧岁月。

这两台机器正式开启了我折腾机器的大门，

回想起来，折腾的原动力来自

1.新鲜感

2.流畅,省电(当时抛开基于cm的rom，谈流畅都相对玄学，许多rom作者修修补补加点小功能精简一点自带app就敢说大幅优化了流畅度，省电)

3.美化（为了状态栏一个图标，也会去单独下个补丁，为了某区域的颜色，能用工具调几个小时）

总结起来就是，追求好用，漂亮与新鲜感

此后的 lg optimus G（G2他爸，G3他爷爷），M8，一加二也都是为此

但从M8开始，我的折腾之路有了些许变化

两颗boomsound扬声器，配合杜比5.1音效，外放至今难有机器超越，后置一颗400w的UltraPixel外加一颗景深镜头，这些特性导致使用第三方rom时，实力都无法展现，因为例如相机算法，音效等属于厂商的商业秘密，绝对竞争力，是绝无可能公开的，所以导致除了官改的rom，第三方rom几乎都无法发挥一部手机的全部特性，这几乎就让我对一大半rom失去了兴趣。

而到了一加二，我使用官方rom的时间就开始远长于第三方rom，为了指纹，为了相机算法，为了稳定性 那些快一点，省电一点又能算得上什么。

当然，第三方rom可不是玩机的全部，我们还有广阔的app，框架，插件，补丁等等资源。xposed框架几乎撑起了折腾的半边天，绿色守护，阻止运行，老牌的滑动返回，重力工具箱，甚至保证你分分钟退群的红包助手。但这个盛夏也在16年迎来寒冬，Android 7.0发布，定名为“Nougat”（牛轧糖），至此Xposed在新系统上再也没能像以往一样归来（类似于ios越狱工具，Xposed非常底层，所以每次新系统更新后其总需要一点时间来支持，但从未太久）

Xposed相对于使用7.0的我们宣布终结。没事，你对我说，我们还有广大的app，绿色，破解，国外开发者，这些不一样美好么

抛开这些不谈。

 

自15年左右开始，我看到了各个厂商定制安卓，促进生态和谐发展的曙光，MIUI越战越勇，功能性易用性 TOp2当之无愧，Flyme也开始摆脱了Bugme的称呼稳步成长，EMUI，FUI等此时还相对沉寂不温不火。而到了如今，连三桑的Touch WIZ都开始加入了主题商店，管理中心等等符合国情的功能，可以窥见 国内厂商的rom到了何种水平。

三、总结

为摆脱硬件软件的同质化，各家发力越来越猛，从人性化的rom到类似于双摄，虹膜，压感，握感（U11）等特有属性，都越发的使得人们难以离开，也失去理由离开官方rom。

我便是这样

Windows：

改DNS，提高上网体验

2016-12-29

  1. DNS是什么

简单来说，我们所访问的在线资源都是服务器的内容，而服务器的地址是一串IP（例如192.168.1.1），IP地址不好记忆，所以有了域名，DNS即是将域名解析到对应IP的一个服务。

就像早期接线员帮我们接入到对应的电话一样。

扩展一下，早期访问互联网内容都是通过host来解析，但随着互联网的发展，在线内容越来越多，host已经完全无法满足需要，所以有了目前这个模式，全球一共有13个DNS服务器，只有一个位于亚洲（日本），目前个人，或公司的DNS服务都类似于此树状图的子节点，13个DNS服务器根为根节点。

2.为何要改DNS

各地运营商均有自己的DNS服务器，小运营商会为了利益，劫持用户DNS，以达到域名劫持，修改注册信息，劫持解析结果来获利。最简单的例子，你输错网址的时候会弹出一堆小广告，

以及宽度快到期时，网页会弹出提醒缴费的页面等等。

而使用靠谱的第三方的DNS服务可以达到：高速，稳定，无广告等作用。

 

3.如何更改DNS

找到你目前所使用的连接，博主目前使用的是wifi，所以从WLAN的熟悉中改，如果是有线网，可能需要从本地连接属性中来改

改这里，然后一路确认就好。

$dir = 'wj';
